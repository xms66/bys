# 同花顺 App 热榜前50贝叶斯决策面板设计

## 目标

用截图口径的同花顺 App 热榜作为唯一真实候选池：热股、大家都在看、1小时。系统返回前50只股票，并展示每只股票的 T+1 买入赚钱概率和贝叶斯决策过程。

## 数据源

真实热榜接口：

`https://dq.10jqka.com.cn/fuyao/hot_list_data/out/hot_list/v1/stock`

请求参数固定为：

- `stock_type=a`
- `type=hour`
- `list_type=normal`

该接口返回的排序与截图一致，例如华能蒙电、大唐发电、长电科技等。接口字段包含股票代码、名称、标签和榜单顺序。行情、换手率、量比、成交额、市值继续用现有腾讯行情补齐。

## 后端行为

新增 `ThsAppHotRankDataSource`。它负责：

1. 拉取同花顺 App 热榜前50。
2. 保留同花顺原始排名。
3. 解析概念标签和人气标签。
4. 用腾讯行情补齐贝叶斯模型需要的量价字段。
5. 没有行情补齐时仍返回热榜条目，并明确缺失量价字段为 0。

`create_data_source()` 默认优先使用 `ThsAppHotRankDataSource`，TickerLab 源只保留为备用或旧兼容，不再作为默认真实同花顺前50。

## 贝叶斯输出

`/api/analysis?top_n=50` 返回：

- `buy_prob`: T+1 买入赚钱概率
- `prior`: 赚钱先验
- `evidence_detail`: 各证据在赚钱/亏钱条件下的似然
- `features`: 热榜排名、概念传播度、K线分、量价活跃度、换手率、量比
- `decision_steps`: 从先验到后验的可展示决策步骤

概率和交易排序分继续分离。`buy_prob` 是后验概率，`signal_score` 是交易排序偏好。

## 前端面板

首页改成工作台式三栏：

1. 左侧：同花顺热榜前50，显示排名、股票、涨跌幅、热度标签、概念标签。
2. 中间：贝叶斯 T+1 赚钱概率列表，显示概率条、信号、排名分。
3. 右侧：选中股票的贝叶斯决策过程，展示先验、市场热度、周期阶段、热榜排名、概念传播、K线形态、量价活跃度和最终后验概率。

保留顶部市场状态卡片，但把文案改成正常中文。

## 测试

先写失败测试：

1. 数据源必须请求同花顺 App 热榜接口和固定参数。
2. 数据源解析出的前几名必须保留热榜顺序和标签。
3. API 必须默认使用 `ths_app_hot_rank` 源。
4. `/api/analysis` 必须返回 `decision_steps` 和前50支持。

最后运行语法检查、pytest，并启动服务验证 `/api/hot_rank?top_n=50` 前几名与截图一致。

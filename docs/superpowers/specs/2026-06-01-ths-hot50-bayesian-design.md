# 同花顺热度前50贝叶斯短线策略设计

## 目标

系统以同花顺热度前50股票作为候选池，估计每只股票在买入后 `T+1 收盘收益 > 0` 的概率：

`P(赚钱 | 当前市场热度, 市场周期, 同花顺热度排名, 概念传播度, K线形态, 量价活跃度)`

输出结果需要是可解释的概率，而不是单纯打分。

## 数据流

1. 获取同花顺热度前50股票，保留热度排名。
2. 补充实时行情字段：价格、涨跌幅、换手率、量比、成交额、流通市值。
3. 提取市场状态：市场热度、上涨/下跌数量、涨停/跌停数量、周期阶段。
4. 提取个股特征：热度排名分、概念传播度、K线形态分、量价活跃度分。
5. 使用朴素贝叶斯计算后验概率。
6. 单独计算交易排序分，避免把概率和偏好权重混在一起。

## 贝叶斯模型

第一版使用规则化朴素贝叶斯：

`posterior = profit_numerator / (profit_numerator + loss_numerator)`

其中：

`profit_numerator = P(profit) * Π P(evidence_i | profit)`

`loss_numerator = P(loss) * Π P(evidence_i | loss)`

先验 `P(profit)` 默认取配置值，后续可用历史交易结果滚动校准。

## 证据因子

- 市场热度：市场成交、涨跌家数、涨跌停强度综合为 0-1。
- 市场周期：退潮、混沌、上升、主升、高潮五阶段。
- 同花顺热度排名：排名越靠前，热度因子越高，但顶部过热可轻微惩罚。
- 概念传播度：使用概念板块涨幅、上涨占比、领涨股强度估计。
- K线形态：基于涨跌幅、振幅、收盘位置、量比、换手率给出可解释标签和分数。
- 量价活跃度：换手率、量比、成交额共同衡量短线活跃性。

## API

`GET /api/hot_rank` 返回真实同花顺热度候选池和行情字段。

`GET /api/analysis` 返回贝叶斯分析结果，包含：

- `buy_prob`: T+1 赚钱概率
- `signal`: 信号等级
- `rank_score`: 交易排序分
- `evidence_detail`: 每个因子的利润/亏损似然
- `features`: 可展示的原始特征分

## 错误处理

同花顺热度接口不可用时，接口返回明确错误和数据源状态。可以保留 Mock 数据用于离线开发，但不能把 Mock 标成真实同花顺数据。

## 测试策略

先写失败测试覆盖：

1. 热度前50来源必须保留外部榜单排名。
2. 贝叶斯概率与交易排序分分离。
3. 高质量证据组合应给出更高后验概率。
4. 真实数据源失败时不能静默伪装成真实数据。

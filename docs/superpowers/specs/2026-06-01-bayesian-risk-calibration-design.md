# 贝叶斯短线模型校准与风控设计

## 目标

结合短线策略可行性报告，避免系统把同花顺热榜股票简单推成高胜率买入。系统应把贝叶斯概率作为决策辅助，并输出风险控制建议。

## 模型约束

短线热度股的单票 T+1 赚钱概率不应轻易超过 70%。第一版没有 2-3 年历史样本校准，因此需要对朴素贝叶斯原始后验做保守校准：

1. 计算原始朴素贝叶斯后验。
2. 将原始后验向 50% 收缩，降低独立性假设带来的过度自信。
3. 根据市场热度与周期阶段做小幅调整。
4. 对追高、放量大跌、极高换手、跌停附近等情况施加风险惩罚。
5. 将最终概率限制在现实短线区间，默认 30%-68%。

## 风控输出

每只股票输出：

- `risk_level`: low / medium / high
- `action`: 观察、轻仓试错、回避
- `max_position_pct`: 单票建议最大仓位
- `stop_loss_pct`: 建议止损
- `take_profit_pct`: 建议止盈
- `reasons`: 风险原因

## API

`/api/analysis` 在每个分析条目中新增：

- `raw_posterior`
- `calibrated`
- `risk_control`
- `model_notice`

`decision_steps` 保留，但最终一步使用校准后概率。

## 前端

决策面板增加：

- 风险等级
- 建议动作
- 单票最大仓位
- 止损/止盈
- 模型说明

## 测试

1. 极强证据组合的校准后概率不得超过 68%。
2. 强证据仍应高于弱证据。
3. API 返回风险控制与模型说明。
